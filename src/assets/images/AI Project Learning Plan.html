🔗 How Core Python Features Connect to Libraries

1. Functions, Modules, and OOP

These help you organize and reuse code, especially in large projects and libraries.

  • Functions: Libraries like NumPy or Matplotlib are made up of many ready-to-use functions.
    Example:

    python
    import numpy as np
np.mean([1, 2, 3])

  • Modules: Libraries are modules. When you import pandas, you're importing a module (or a package made of modules).

  • OOP (Classes and Objects): Most libraries use classes internally. For example:

    python
    import pandas as pd
df = pd.DataFrame()  # DataFrame is a class

So when you use a library, you're using objects and functions created by someone else.

2. File Handling

Libraries often work with files — so you must understand how file input/output works.

  • Pandas reads data files:

    python
    df = pd.read_csv("data.csv")

  • You might also write to a file yourself:

    python
    with open("output.txt", "w") as f:
    f.write("Hello AI")

Even AI models (e.g., PyTorch) save and load weights from files.

3. Exception Handling

Library functions often throw errors — exception handling helps manage them without crashing your program.

Example:

python
try:
    df = pd.read_csv("nonexistent.csv")
except FileNotFoundError:
    print("File not found.")

This is critical in production AI pipelines, where missing files or bad data shouldn't crash your entire system.

✅ Correct Code Example:

python
import pandas as pd
import matplotlib.pyplot as plt

# Load the CSV file
df = pd.read_csv("students.csv")

# Filter students with score > 80
top_students = df[df["score"] > 80]

# Plot bar chart: names on x-axis, scores on y-axis
plt.bar(top_students["name"], top_students["score"])

# Add labels and title
plt.xlabel("Name")
plt.ylabel("Score")
plt.title("Top Students")

# Show the plot
plt.show()

🛠️ Common Mistake:

You might be doing something like this:

python
plt.bar(top_students["name", "score"])  # ❌

This causes:

TypeError: list indices must be integers or slices, not tuple

Why? Because "name", "score" is a tuple, and you’re trying to access two columns in the wrong way.

✅ The right way:

  • Use top_students["name"] → gives the name column

  • Use top_students["score"] → gives the score column
